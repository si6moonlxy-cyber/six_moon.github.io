# 工作流剖析
>适用人群:使用多工具在Vibe Coding中的开发者,想使用AI工具的小白
>推荐使用场景:学习ai工具的边界;想了解AI工具的使用;更结构化使用AI

---

|工具|身份定位|核心职责|不能让它乱做的事|
|---|---|---|---|
|**Claude**|总架构师 / 产品技术顾问 / 审查官|需求拆解、方案设计、代码审查、重构建议、文档口径统一、复杂问题判断|不直接大量改文件，不做无验证的最终结论|
|**Codex**|执行型工程师 / 任务工人 / PR 生成器|修 bug、写功能、改代码、跑测试、生成变更说明、处理明确任务|不决定产品方向，不随意扩大需求范围|
|**VS Code**|本地驾驶舱 / 项目现场 / 人工验收台|打开项目、查看目录、运行服务、检查 Git diff、手动微调、最终确认|不作为长期知识库，不把临时上下文当记忆|
|**Obsidian**|知识仓库 / 决策档案馆 / 本地记忆中心|存放 PRD、需求、架构、复盘、经验、项目背景、长期约束|不放大量临时代码、不代替 Git 管代码|
|**API 管理工具**|接口合同官 / 联调验证台|管理接口文档、请求示例、环境变量、Mock、接口测试、验收用例|不替代后端测试，不作为业务需求唯一来源|
